# Repository Memory

## Stable Context
- **長期規則與慣例**  
  - 只以 GitHub Issue / Comment 為原始資料來源，手動筆記僅作為摘要與指引，**不會被覆寫**。  
  - 任何 **compact‑memory workflow** 會讀取 `shared/manual.md`，但不會將其內容直接寫回。  
  - 代理（agents）在整理記憶時，**不會逐段複製原始 Issue 文字**，而是抽取可重用的概念與決策。  
  - 需要 **保持 repo 的可讀性與工程可維護性**，因此所有長期記憶皆以精簡、結構化的方式呈現。  

- **目前的資料狀態**  
  - 從 2026‑06‑27 至 2026‑07‑03 的每日快照皆顯示 **無可用 Issue**，因此沒有新增的跨 Issue 主題、決策或可辨識的標籤。  
  - 既有的長期記憶（若有）仍保留於本檔，未因每日快照而被清除或覆寫。

## Constraints
1. **資料來源限制**  
   - 只能從 GitHub Issue / Comment 抽取資訊；手動筆記僅作為輔助說明。  
2. **寫入規則**  
   - 不得直接複製 Issue 原文；必須以「蒸餾」後的概念呈現。  
   - `shared/manual.md` 為只讀參考檔，**不允許自動覆寫**。  
3. **可視化與可讀性**  
   - 所有條目須保持 Markdown 結構，方便程式與人類閱讀。  
4. **不確定資訊處理**  
   - 若資訊不足或相互矛盾，必須在相應節點標註「不確定」或「需進一步驗證」。
