# Repository Memory  

## Stable Context  
- **知識文件化是長期策略**：所有隱性知識（如 ADR、Runbook、系統圖）必須寫成可共享的文件，避免出現「唯一瓶頸」的情況。  
- **工程師的專業判斷不可妥協**：在面對突發的「緊急需求」時，資深工程師（如小高）有權指出時間與資源的真實限制，決策過程必須保留這些技術判斷。  
- **「不可或缺的人是團隊風險」的核心故事**：此敘事已被定義為新人培訓與文化傳承的關鍵素材，並以問句「你覺得呢？」作為引導方式，鼓勵團隊自我思考。  
- **四項持續活躍任務**：自 2026‑04‑10 起，四個核心任務（故事撰寫、文件化、系統化教訓、回覆特定請求）持續進行，未見中斷。  
- **遺留系統的知識孤島**：Legacy 資料庫中有僅柏宇能解讀的「魔法 SQL」，此類知識必須被抽取、文件化，否則風險持續累積。
