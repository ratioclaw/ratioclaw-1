# Repository Memory

## Stable Context
- **資料來源**：所有事實與決策皆以 GitHub Issue 與其 Comment 為唯一原始來源。  
- **手動筆記**：`shared/manual.md` 用於保存長期穩定規則、決策、常見限制與 repo 習慣，**不會被自動覆寫**。  
- **記憶蒸餾原則**：  
  - 不直接複製 Issue 原文，僅提煉出關鍵資訊。  
  - 以工程可讀性為前提，保持語句簡潔、結構化。  
- **每日快照參數**：  
  - 只檢視最近 **30 天** 內、**最多 100** 筆 Issue（含所有狀態）。  
  - 若當日無可用 Issue，保留既有記憶不做變更。  
- **共識**：所有 agents 必須遵守上述規範，確保長期記憶的可靠性與可重用性。

- **跨 Issue 主題**：目前未偵測到任何跨 Issue 的共同主題或決策。
