# Repository Memory

## Stable Context
- **目前無可辨識的長期規則或決策**：過去 30 天的 daily snapshots 均未檢測到任何可用的 issue，故無法抽取持續性的工作流程、程式碼慣例或主人偏好。  
- **手動維護的共享筆記**（`shared/manual.md`）仍是唯一的長期記憶來源，內容僅列出 **「應放入此檔的類型」**（穩定規則、長期決策、常見限制、repo 習慣），但未提供具體實例。  
- **系統狀態**：repo `ratioclaw/ratioclaw-1` 在最近 30 天內未產生任何可供 agents 處理的 issue，所有 daily snapshots 均顯示「本次整理視窗沒有可用 issue」。

> **不確定性**：因缺乏實際 issue 資料，無法確認主人是否有隱藏的工作流程或未記錄的限制。若未來出現 issue，請立即更新此段落。

## Recent Themes
- **無跨 issue 主題**：每日快照皆報告「目前沒有可辨識的跨 issue 主題」。  
- **等待新資訊**：所有快照的 **Open Loops** 都指向「等待下一輪 issue 更新後再整理」，顯示目前的焦點是 **觀察** 而非執行。

## Constraints
- **資料來源限制**：  
  1. 只能以 GitHub issue / comment 為原始資料來源，不能直接引用手動筆記的文字。  
  2. `shared/manual.md` 僅作為 **指引**，不應被視為具體規則。  
- **記憶更新規則**：  
  - 若同一天內出現多筆相同資訊，僅保留一次，避免重複。  
  - 只有在多日持續出現且具備一致性的資訊，才會升級為 **Stable Context**。  

## Open Loops
- **Issue 收集**：目前缺乏可用 issue，需等待新 issue 的產生或現有 issue 的狀態變更。  
- **跨日主題探索**：若未來出現跨多日重複的主題，需將其歸類至 **Recent Themes** 或提升為 **Stable Context**。  
- **手動筆記補充**：需要主人或管理者在 `shared/manual.md` 中加入具體的長期規則、決策或限制，以便龍蝦們有可供蒸餾的實質內容。
