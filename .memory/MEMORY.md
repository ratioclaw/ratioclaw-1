# Repository Memory

## Stable Context
- **長期規則與決策**  
  - 以 **GitHub Issue / Comment** 為唯一原始資料來源，所有記憶皆應根據這些訊息產生。  
  - `compact-memory` 工作流程會 **讀取** 本手動筆記，但 **不會覆寫**，因此本檔案僅作為長期參考，不會被自動更新。  
  - 不得直接複製完整的 Issue 內容於此檔案；僅保留 **精煉、抽象化** 的資訊（如規則、限制、共識）。  

- **Repo 共同習慣**  
  - 所有 agents 必須遵守上述來源與寫作原則。  
  - 每日快照（daily snapshots）會在 **過去 30 天** 內檢視所有 Issue，若無可用 Issue，則保留既有記憶不變。  

- **已確認的長期事實**  
  - 目前在 `ratioclaw/ratioclaw-1` 的最近 30 天內，**沒有任何可用 Issue** 被偵測到。  
  - 因此，除手動筆記外，沒有其他可驗證的長期資訊可加入本記憶庫。

> **註**：若未來出現新的 Issue，相關主題將在此區塊更新。

## Constraints
1. **資料來源限制**  
   - 只能根據 GitHub Issue / Comment 產生記憶；其他來源（如聊天紀錄、外部文件）不列入本 repo 的正式記憶。  
2. **內容寫作限制**  
   - 禁止在本檔案中直接貼上完整 Issue 文字。  
   - 必須以 **精煉、可重用** 的形式呈現，避免冗長或重複。  
3. **更新機制**  
   - `compact-memory` workflow 只會 **讀取** 本手動筆記，**不會自動覆寫**。任何變更須由人類手動編輯。
