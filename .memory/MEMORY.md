# Repository Memory

## Stable Context
- **手動筆記原則**  
  - 只在 `shared/manual.md` 中保留 **穩定規則、長期決策、常見限制、repo 習慣**。  
  - 原始資料來源必須是 GitHub Issue / Comment，手動筆記不會被自動覆寫。  
  - 任何內容 **不應直接複製** 完整的 Issue 原文，而是以精煉、可重用的形式呈現。  

- **目前觀測**  
  - 最近 30 天的 daily snapshots（2026‑06‑02 ~ 2026‑06‑08）皆未偵測到可用 Issue，故 **沒有新增的穩定規則或決策** 可加入此段落。  

> **不確定性**：因缺乏 Issue 資料，無法確認是否存在尚未被捕捉的長期規則或決策。若未來出現新 Issue，請於此段落補充。

---

## Recent Themes
- **無跨 Issue 主題**：每日快照皆報告「目前沒有可辨識的跨 issue 主題」。  
- **無新決策**：每日快照皆顯示「目前沒有新的跨 issue 決策」。  

> **觀察**：近期（過去一週）沒有任何重複出現的主題或討論點可歸納為「近期主題」。

---

## Constraints
- **資料來源限制**  
  - 只能從 GitHub Issue / Comment 取得原始資訊，手動筆記僅作為摘要與規則的存放地。  
  - `shared/manual.md` 不會被自動流程覆寫，必須由人類手動維護。  

- **內容呈現限制**  
  - 不得在 `MEMORY.md` 中逐段複製原始 Issue 文字。  
  - 必須將資訊蒸餾為 **可重用、工程可讀** 的敘述。  

- **流程限制**  
  - `compact-memory workflow` 只會讀取 `shared/manual.md`，不會寫入或修改它。  

---

## Open Loops
- **等待 Issue 更新**  
  - 每日快照皆列出「等待下一輪 issue 更新後再整理」，表示目前 **沒有待處理的 Issue**。  
  - 若未來有新 Issue 出現，需重新評估是否產生以下類型的開放環節：  
    - 新的跨 Issue 主題  
    - 未決策的議題  
    - 需要持續追蹤的限制或規則  

- **資訊不足**  
  - 目前缺乏任何具體的
