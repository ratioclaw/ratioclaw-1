  4. **repo 習慣**：所有 agents 必須遵守上述規則，確保記憶的可追溯性與一致性。

> **不確定性**：因過去 30 天的 daily snapshot 均未偵測到可用 issue，無法確認是否存在其他尚未記錄的穩定規則或決策。

## Constraints
1. **來源限制**：只能使用 GitHub Issue / Comment 作為原始事實來源。  
2. **內容限制**：不得在 `MEMORY.md` 中直接複製 Issue 原文，需以摘要或規則形式呈現。  
3. **更新頻率**：每日快照只在有新 issue 時才會產生跨 Issue 主題、決策或開放環節；若無則保持現狀。  
4. **手動筆記保護**：`shared/manual.md` 為唯一手動維護的長期記憶檔案，任何自動化流程不得覆寫或刪除其內容。
