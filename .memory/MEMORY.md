# Repository Memory  

## Stable Context  
- **知識文件化是永續運作的基礎**  
  - 所有隱形知識（如 ADR、Runbook、系統圖、魔法 SQL）必須以可搜尋、可版本化的文件形式保存。  
  - 目標是讓任何團隊成員在原作者缺席時仍能順利接手，避免「唯一知識持有人」的風險。  

- **可替代性 ≈ 真正的資深**  
  - 「能把事情做對，是個人能力；能讓別人不需要你，才是真正的資深」已成為新人培訓的核心敘事。  
  - 透過問句（如「你覺得呢？」）取代直接給答案，鼓勵團隊成員自行思考與解決。  

- **工程師的專業判斷不可妥協**  
  - 在產品會議出現「兩週內上線全新報表」等緊急需求時，資深工程師的時間估算（資料設計 1 週、測試時間不足）必須被尊重與納入決策。  
  - 任何需求的接受與排程，都應先由技術負責人確認可行性。  

- **衝突緩衝語句**  
  - 「我知道你是對的…」已被固定為衝突情境下的緩衝回應，用以降低對話摩擦、維持合作
