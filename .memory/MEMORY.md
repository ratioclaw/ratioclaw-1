# Repository Memory

## Stable Context
- **長期規則與決策**  
  - 以 GitHub Issue / Comment 為唯一的原始資料來源，所有可追溯的事實與決策必須能在這些記錄中找到對應。  
  - `shared/manual.md` 為人工維護的長期記憶檔案，僅存放 **穩定規則、長期決策、常見限制** 以及 **repo 共同習慣**，不會被自動流程覆寫。  
  - 任何自動化的 `compact‑memory` 工作流會 **讀取** 這份手動筆記，但 **不會寫入**，因此手動筆記的內容始終保持不變。  
  - 產出給 agents 的長期記憶（本文件）必須是 **精煉、可重用** 的敘述，避免逐段複製原始 Issue 文字。  

- **Repo 習慣**  
  - 所有 issue 必須在 30 天內被 agents 掃描；若超過此期限且仍未出現，視為「暫無可用 issue」。  
  - 每日快照只會列出 **跨 Issue 主題、決策與未完成事項**，若無資料則保留既有記憶不變。  

## Recent Themes
- **近期無可辨識的跨 Issue 主題**  
  - 從 2026‑05‑26 至 2026‑06‑01 的每日快照皆顯示「本次整理視窗沒有可用 issue」，因此目前沒有重複出現的主題可供蒸餾。  

- **持續的空白狀態**  
  - 每日快照的 `Open Loops` 均為「等待下一輪 issue 更新後再整理」，顯示目前的工作流正處於待命狀態，未產生新議題或決策。  

## Constraints
1. **資料來源限制**  
   - 只能引用 GitHub Issue / Comment 作為事實依據；任何其他來源（如聊天記錄、外部文件）不應被視為正式記憶。  

2. **內容重複與版權**  
   - 不得完整複製 Issue 原文；必須以 **摘要** 或 **概念化** 的方式呈現。  

3. **手動筆記保護**  
   - `shared/manual.md` 只接受人工編輯，任何自動化流程（包括本文件的生成）都不會修改它。  

4. **時間範圍**  
   - 只考慮過去 30 天內的 issue 活動；超出此範圍的資訊若未在快
