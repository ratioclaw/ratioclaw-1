# Repository Memory

## Stable Context
- **長期規則與決策**  
  - 依據 *Shared Memory Manual Notes*，本倉庫的長期規則、決策與常見限制皆應記錄於手動維護的 `shared/manual.md` 中。  
  - 任何 **穩定規則**（例如程式碼風格、部署流程、分支策略）應在此檔案內明確說明，供所有 agents 共同遵守。  

- **Repo 習慣**  
  - 所有 issue、comment 為 **唯一原始資料來源**，Agents 只應從這些結構化資訊抽取事實，絕不直接複製完整的 issue 文字於記憶檔。  
  - `compact‑memory` 工作流程會讀取 `shared/manual.md`，但 **不會覆寫**，因此手動筆記的內容必須保持最新且具可讀性。  

- **目前狀態**  
  - 從 2026‑06‑30 至 2026‑07‑06 的每日快照皆顯示 **無可用 issue**，因此沒有新增的跨 issue 主題、決策或可供蒸餾的資訊。  
  - 既有的長期記憶（若有）仍保留於本檔案或 `shared/manual.md`，未因缺乏新 issue 而被刪除或覆寫。

## Recent Themes
- **無可辨識的跨 issue 主題**  
  - 連續七天的快照皆未偵測到任何跨 issue 的共通主題或重複出現的討論。  
  - 因此本節暫時保持空白，待未來有可觀測的主題時再行填寫。

## Constraints
1. **資料來源限制**  
   - 只能從 GitHub Issue / Comment 中抽取資訊；手動筆記僅作為補充說明，不能作為唯一事實來源。  
2. **避免冗餘**  
   - 不得將原始 issue 文字逐段複製至本記憶檔，必須以 **摘要、規則或決策** 形式呈現。  
3. **更新頻率**  
   - 每日快照若無新 issue，則保留既有記憶，不進行任何刪除或重寫。  
4. **不確定資訊標示**  
   - 若資訊不足或相互矛盾，必須在相應節點標註「不確定」或「需進一步驗證」。
