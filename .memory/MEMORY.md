# Repository Memory

## Stable Context
- **資料來源原則**  
  - 只將 **GitHub Issue / Comment** 視為原始事實來源。  
  - `shared/manual.md` 只保存 **長期規則、決策、限制與共識**，不會被自動覆寫。  
  - 任何自動化流程（如 *compact‑memory*）會 **讀取** 本手動筆記，但 **不會寫入**。

- **記憶整理慣例**  
  - 每日快照只彙整當天活躍的 issue agents，若無可用 issue，則保留既有記憶不變。  
  - 跨 Issue 的主題、決策與開放環節（Open Loops）僅在有實質資料時才加入長期記憶。  
  - 任何 **暫時性**、僅在單一天出現且未形成穩定事實的資訊，應歸入 **Recent Themes** 或 **Open Loops**，絕不寫成永久規則。

- **語氣與角色**  
  - 本記憶由一群忠誠的 **龍蝦** 代主人體現，語調溫暖且具工程可讀性，確保主人在繁忙時仍能快速抓取關鍵脈絡。

## Recent Themes
- **缺乏活躍 Issue**  
  - 2026‑05‑11 至 2026‑05‑17 的每日快照皆顯示「本次整理視窗沒有可用 issue」，說明近期 **repo 內無新議題** 或 **agents 未偵測到待處理項目**。  
  - 這是一個持續的觀測結果，暗示目前的開發或維護工作可能處於靜止、等待新需求或是已完成階段。

- **無跨 Issue 主題**  
  - 每日快照均報告「目前沒有可辨識的跨 issue 主題」，表示 **尚未出現需要跨多個 Issue 協調的共通議題**。

- **決策與標籤**  
  - 同樣地，近期未產生新的跨 Issue 決策，且 **Top Labels** 為 `none`，顯示目前的 Issue 標籤系統未被使用或未有新標籤加入。
