# Repository Memory

## Stable Context
- **長期規則**  
  - 只以 GitHub Issue / Comment 為原始資料來源，任何摘要或記錄皆須根據這些來源抽取。  
  - `shared/manual.md` 為唯一的手動維護長期記憶檔案，**不會**被自動流程覆寫。  
  - 任何自動化流程（如 compact‑memory）只能**讀取**此手動筆記，不能寫入或改寫。  
  - 內容必須**精簡**、避免逐段複製原始 Issue 文字；重點在於抽象出規則、決策、限制與未完成事項。  

- **Repo 習慣**  
  - 所有 agents 必須在每日快照（daily snapshots）中報告：  
    1. 本日可用 Issue 數量與狀態  
    2. 跨 Issue 主題（若有）  
    3. 新的跨 Issue 決策（若有）  
    4. 開放迴路（Open Loops）  
  - 若當日無可用 Issue，則保留既有記憶並在「Open Loops」中標註等待下一輪更新。  

- **長期決策**  
  - 目前尚未出現任何跨 Issue 的決策，故無固定決策可列入長期記憶。  

## Recent Themes
- **缺乏可用 Issue**  
  - 2026‑07‑09 至 2026‑07‑15 的每日快照皆顯示「本次整理視窗沒有可用 issue」，表示近期 repo 內的 Issue 活動極低或已全部關閉。  
- **無跨 Issue 主題**  
  - 每日報告均未偵測到可辨識的跨 Issue 主題，說明目前沒有需要跨多個 Issue 共同處理的議題。  

> **註**：上述主題僅基於最近七天的快照，若未來出現新 Issue，主題可能會快速變化。

## Constraints
1. **資料來源限制**  
   - 只能引用 Issue / Comment 作為事實依據，任何推論必須標示為「不確定」或「待驗證」。  
2. **手動筆記保護**  
   - `shared/manual.md` 必須保持原樣，任何自動化產出（如本文件）不得寫入或覆寫此檔。  
3. **摘要原則**  
   - 不得逐段複製原始 Issue 文字；必須將資訊蒸餾成可重用的規則或上下文。
