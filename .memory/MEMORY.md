# Repository Memory

## Stable Context
- 本倉庫的長期記憶由 **shared/manual.md** 內的手動筆記維護，主要保存 **穩定規則、長期決策、常見限制** 以及 **agent 共同遵守的 repo 習慣**。  
- 所有自動化的記憶來源皆為 **GitHub Issue / Comment**，這些原始資料在日常快照中被 agents 讀取、彙整。  
- **compact‑memory workflow** 會在每次執行時讀取 `shared/manual.md`，但不會覆寫此檔案，確保手動編寫的規則永遠保留。  
- 目前 **沒有任何活躍的 Issue**（過去 30 天內皆未出現可用 Issue），因此系統未產生新的跨 Issue 主題或決策。

> **不確定性**：因缺乏 Issue 資料，無法確認是否存在隱藏的長期規則或未記錄的限制。若未來出現 Issue，需重新評估並補充至此節。

## Recent Themes
- **無可辨識的跨 Issue 主題**：過去一週（2026‑07‑12 至 2026‑07‑18）的所有快照皆顯示「目前沒有可辨識的跨 issue 主題」。
- **無新決策**：同樣的時間範圍內未出現任何跨 Issue 決策。

> **觀察**：近期的「無資訊」本身是一個主題，代表倉庫目前處於靜止或待命狀態，等待新任務或 Issue 的產生。

## Constraints
1. **資料來源限制**  
   - 只能從 **GitHub Issue / Comment** 取得原始事實。若 Issue 被關閉或刪除，相關記憶將不再可用。  
2. **手動筆記不可被自動流程覆寫**  
   - `shared/manual.md` 只作為長期規則的「只讀」來源，任何自動化的記憶整理不會改寫此檔。  
3. **快照窗口限制**  
   - 每次日誌快照僅檢視最近 30 天內的 Issue，超出此範圍的資訊不會自動納入本記憶體。  

> **不確定性**：若未來有外部資料（如 PR、外部文件）需要納入長期記憶，現行流程尚未定
