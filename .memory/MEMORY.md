  - GitHub Issue / Comment 為原始資料來源，手動筆記僅作為摘要與指引。  
  - `compact‑memory` 工作流程會讀取此手動筆記，但不會覆寫它。  

- **目前觀測到的長期事實**  
  - 近期 30 天內的所有 daily snapshot 均未偵測到可用 Issue，表示 **目前 repo 中沒有待處理或新建立的 Issue**。  
  - 因缺乏具體 Issue，無法從日誌中抽取額外的穩定規則或決策。  

> **不確定性**：除上述手冊規則外，沒有其他可驗證的長期事實可供記錄。若未來出現新的 Issue，將會補充相應的穩定上下文。

## Constraints
1. **資料來源限制**  
   - 只能從 GitHub Issue / Comment 取得原始資訊，手動筆記僅作為輔助。  
2. **內容編寫規範**  
   - 不得完整複製 Issue 文字；必須以 **精煉、可重用** 的方式呈現。  
   - `MEMORY.md` 必須保持 **長期可讀性**，避免雜訊與臨時資訊。  
3. **更新節奏**  
   - 若每日快照未偵測到 Issue，則暫停新增記憶，僅保留現有內容。  

## Open Loops
