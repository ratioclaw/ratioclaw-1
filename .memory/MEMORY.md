# Repository Memory

## Stable Context
- **資料來源**：所有長期記憶皆應以 GitHub Issue 或 Comment 為原始依據，手動筆記僅作為輔助摘要，**不會被自動覆寫**。  
- **compact‑memory 工作流程**：會讀取 `shared/manual.md` 內的手動筆記，以提供穩定規則與限制，然而不會將其內容直接寫回原始檔案。  
- **Issue 觀測窗口**：目前 30 天內的 Issue 列表皆為空，代表近期沒有新建立或更新的工作項目。  

> **不確定性**：因為缺乏具體 Issue 記錄，無法確認是否存在隱藏的長期決策或規則。若未來出現新 Issue，請於此節補充。

## Recent Themes
- **無可辨識跨 Issue 主題**：連續七天的每日快照皆未偵測到任何跨 Issue 的共通主題或重複議題。  
- **暫無新決策**：近期未產生新的跨 Issue 決策。  

> 若未來出現重複出現的議題（例如持續的 bug、功能需求或流程改進），請將其歸納於此節。

## Constraints
- **手動筆記限制**：  
  - 只能放置 **穩定規則、長期決策、常見限制、repo 習慣**。  
  - 不得直接複製 Issue 原文。  
- **資料完整性**：  
  - 所有正式的事實與決策必須以 Issue / Comment 為根據，手動筆記僅作為摘要。  
- **更新頻率**：  
  - 代理人每日會檢查過去 30 天的 Issue，若無新資料則保留既有記憶。  

## Open Loops
- **等待 Issue 更新**：目前所有每日快照皆顯示「等待下一輪 Issue 更新後再整理」，因此所有未完成的工作項目、待決策事項或潛在問題皆處於待觀察狀態。  
- **潛在待辦**：若在未來的 Issue 中出現新需求或問題，請將其列於此節，並在後續快照中追蹤其進度。
