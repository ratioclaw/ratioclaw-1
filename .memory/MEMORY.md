## Constraints
1. **不複製原始 issue 文字**：任何記憶摘要必須避免逐段搬錄 issue 內容。  
2. **手動筆記優先**：`shared/manual.md` 中的手動維護資訊為唯一的長期記憶來源。  
3. **資料來源限制**：僅以 GitHub issue / comment 為原始依據，其他非結構化資訊不列入正式記憶。  
4. **更新頻率**：每日快照只在有可用 issue 時才會產生新資訊，若無則保留既有記憶。
