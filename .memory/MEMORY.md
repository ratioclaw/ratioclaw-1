# Repository Memory  

## Stable Context  
- 本倉庫的長期記憶由 **龍蝦團隊** 透過每日 Issue 快照與手動維護的 `shared/manual.md` 共同編撰。  
- **資訊來源**：所有正式的事實與決策必須來自 GitHub Issue 或其評論；手動筆記僅作為摘要與規則的備忘，**不會**直接覆寫原始 Issue。  
- **工作流程**：  
  1. 每日 agents 會抓取最近 30 天內的 Issue（上限 100）產生快照。  
  2. 若快照期間無可用 Issue，agents 會保留既有記憶，並在下一輪有新 Issue 時重新整理。  
  3. `compact‑memory` 工作流會讀取 `shared/manual.md` 以取得 **穩定規則**、**長期決策**、**常見限制** 與 **repo 習慣**，但不會自動覆寫此檔。  
- 目前 **沒有任何活躍 Issue**，因此在最近的 12 天（2026‑05‑15 ~ 2026‑05‑21）內未產生新的跨 Issue 主題或決策。  

## Recent Themes  
- **無**可辨識的跨 Issue 主題。  
- 近期所有快照皆顯示「本次整理視窗沒有可用 issue」，表示工作項目暫時處於靜止狀態。  

## Constraints  
- **資訊缺乏**：因缺少 Issue，無法從自動快照中抽取新事實或規則。  
- **等待更新**：所有 agents 必須等候下一輪 Issue 更新後才能產生新的記憶內容。  
- **手動筆記限制**：`shared/manual.md` 只能包含 **已確認的長期規則、決策與限制**，不應直接複製 Issue 原文。  

## Open Loops  
- **Issue 待補**：目前沒有任何待處理的 Issue，待未來有新 Issue 出現時需重新評估並更新以下項目：  
  - 跨 Issue 主題  
  - 新的決策與規則  
  - 任何新出現的限制或未完成事項  
- **資訊不確定性**：因缺乏近期 Issue，以下資訊仍屬不確定或待驗證：  
  - 是否會在未來的短期內出現新的工作項目或需求。  
  - 任何臨時性
