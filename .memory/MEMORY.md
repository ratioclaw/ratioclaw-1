# Repository Memory

## Stable Context
- **長期規則與決策**  
  - 以 GitHub Issue 與 Comment 為唯一的原始資料來源，所有記憶的抽取與整理必須根據這些條目。  
  - `compact-memory` 工作流程會讀取 **shared/manual.md** 中的手動筆記，但不會對其進行覆寫。  
  - 任何 **stable rule**（穩定規則）皆應寫入 **shared/manual.md**，而非每日快照或臨時筆記。  

- **Repo 習慣**  
  - 只保留 **精煉後** 的長期可重用資訊，避免逐段複製原始 Issue 內容。  
  - 每日快照僅用於偵測跨 Issue 主題、決策與未完成事項，若無新資訊則保留既有記憶。  

- **限制**  
  - 不得在 **MEMORY.md** 中直接貼上 Issue 原文或完整的討論內容。  
  - 只能呈現「規則、決策、限制、未完成事項」等抽象層面的資訊。  

## Recent Themes
- **無可辨識的跨 Issue 主題**  
  - 2026‑05‑21 至 2026‑05‑27 的每日快照皆顯示「目前沒有可辨識的跨 issue 主題」。  
- **缺乏新決策**  
  - 同期間內未出現任何跨 Issue 的新決策。  

> **註**：因為連續七天皆未檢測到可用 Issue，近期主題暫時為「資訊缺乏」與「等待新 Issue」的狀態。

## Constraints
1. **資料來源限制**  
   - 只能從 GitHub Issue / Comment 取得資訊；其他來源（如聊天記錄、外部文件）不列入正式記憶。  
2. **編輯限制**  
   - `shared/manual.md` 為手動維護的長期記憶檔案，系統不會自動覆寫或修改其內容。  
3. **呈現限制**  
   - `MEMORY.md` 必須保持可讀性與工程友好性，避免冗長的原文堆砌。  
4. **不確定性處理**  
   - 若資訊不足或相互矛盾，必須在相應節點標註「不確定」或「需進一步驗證」。
