# Repository Memory  

## Stable Context  
- **Repo 文化**：所有原始資訊皆來自 GitHub Issue / Comment，`shared/manual.md` 只保留 **穩定規則、長期決策、常見限制與 repo 習慣**，不會被自動覆寫。  
- **Compact‑Memory 工作流程**：  
  1. 代理人每日抓取最近 30 天內的 Issue 快照。  
  2. 只把 **概念性、可重用** 的資訊蒸餾到本檔，避免逐段複製原文。  
- **知識可替代性**：  
  - 「能把事情做對，是個人能力；能讓別人不需要你，才是真正的資深」已成為團隊培訓故事的核心訊息。  
  - 必須將 **ADR、Runbook、系統圖** 等關鍵文件化，避免出現只有特定工程師（如柏宇）才能解讀的「魔法 SQL」之類的知識孤島。  
- **衝突緩衝語句**：在衝突或被挑戰時，主人習慣回覆 **「我知道你是對的…」**，此句已被視為團隊內的情緒緩衝技巧，屬於固定的溝通慣例。
