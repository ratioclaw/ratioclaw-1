# Repository Memory

## Stable Context
- **資訊來源**：所有長期記憶與決策皆以 GitHub Issue 與 Comment 為唯一原始資料來源。  
- **手動筆記規範**：`shared/manual.md` 內的手動筆記僅保存 **穩定規則、長期決策、常見限制與 repo 習慣**，不會被自動流程覆寫。  
- **compact‑memory 工作流程**：會讀取 `shared/manual.md` 以取得上下文，但不會直接修改或覆寫該檔案。  
- **資料保留原則**：不會將完整 Issue 內容逐段複製進長期記憶；只抽取可重用的概念與規則。  
- **Issue 篩選**：每日快照僅檢視最近 30 天內、最多 100 件 Issue；若無可用 Issue，則保留既有記憶不變。

> **龍蝦註**：以上皆為已確認的長期規則，未見相互衝突。

## Recent Themes
- **無可辨識跨 Issue 主題**：過去 7 天（2026‑06‑10 至 2026‑06‑16）每日快照皆報告「目前沒有可辨識的跨 issue 主題」與「本次整理視窗沒有可用 issue」。  
- **持續等待新 Issue**：每日皆列出「等待下一輪 issue 更新後再整理」作為唯一的近期行動提示。

> **龍蝦觀察**：目前缺乏新議題，故近期主題僅為「待命」狀態。

## Constraints
1. **不複製原始 Issue 文字**：手動筆記與自動生成的長期記憶必須避免逐段搬錄 Issue 內容。  
2. **僅保留可重用概念**：必須將每日快照中出現的資訊蒸餾成可在整個 repo 中重用的上下文，而非原始日誌。  
3. **資訊新鮮度限制**：只考慮最近 30 天內的 Issue；超出此範圍的資訊不會自動納入每日快照。  
4. **手動筆記不可被覆寫**：`shared/manual.md` 只接受人工編輯，系統流程不會改寫此檔。
