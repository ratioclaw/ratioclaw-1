# Repository Memory  

## Stable Context  
- 目前 **沒有** 從 issue 中觀測到可確認的長期規則、決策或慣例。  
- 依照 **Shared Memory Manual Notes**，穩定規則、長期決策與常見限制應由人類手動維護於 `shared/manual.md`，但此檔案目前僅包含說明文字，未列出具體內容。  
- 因缺乏可驗證的資料，我們只能將「無可用 issue」視為暫時的 **空白狀態**，而非一項規則。  

## Recent Themes  
- **2026‑05‑28 至 2026‑06‑03** 的每日快照皆顯示「本次整理視窗沒有可用 issue，先保留既有記憶」。  
- 沒有跨 issue 的主題、決策或標籤被辨識出來。  
- 因此近期沒有可重複出現的主題可蒸餾為長期記憶。  

## Constraints  
- **資訊來源限制**：所有原始事實必須來自 GitHub issue / comment。若未出現在 issue 中，則不應被視為確定事實。  
- **手動筆記不可覆寫**：`shared/manual.md` 會被 `compact‑memory` 工作流程讀取，但不會被自動覆寫。  
- **不確定性聲明**：當資料不足或相互矛盾時，必須明確標示為「不確定」或「缺乏資訊」。  

## Open Loops  
- **等待新 Issue**：所有每日快照皆將「等待下一輪 issue 更新後再整理」列為唯一的開放環節。  
- **未確定的長期規則**：因目前尚未觀測到任何穩定規則，未來若出現持續出現的 issue（例如代碼風格、部署流程等），需重新評估並加入 **Stable Context**。  
- **手動筆記補充**：若人類在 `shared/manual.md` 中加入具體的穩定規則或限制，需同步更新本文件的相應章節。  

> **龍蝦的提醒**：目前的記憶庫如同空的潮汐池，只有海浪（新 issue）來臨時才會留下足跡。請持續提交 issue，讓我們的長期記憶得以逐漸填滿。
