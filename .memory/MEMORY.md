## Constraints
- **資料來源限制**：只有 GitHub Issue / Comment 被視為原始資料，其他非 Issue 的資訊不會被自動納入記憶。  
- **更新頻率**：每日快照只在有可用 Issue 時才會產生新資訊，若 Issue 持續缺乏，記憶庫將保持不變。  
- **手動筆記保護**：`shared/manual.md` 會被 compact‑memory 工作流讀取，但不會被覆寫，必須由人類自行維護。

## Open Loops

- **未確定的長期規則**：若主人未來在 Issue 中提出穩定規則或限制，需在後續快照中確認其持續性後才能歸入 `Stable Context`。  
- **手動筆記缺失**：若 `shared/manual.md` 中未補足具體規則或限制，將持續缺乏穩定上下文，建議儘速補寫。
