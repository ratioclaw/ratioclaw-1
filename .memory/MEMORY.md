# Repository Memory

## Stable Context
- **長期規則**  
  - 只以 GitHub Issue / Comment 為原始資料來源，任何摘要或記錄皆須基於這些正式條目。  
  - `shared/manual.md` 為手動維護的長期記憶檔案，**不會**被自動流程覆寫。  
  - 任何自動化的「compact‑memory」工作流只能**讀取**此手動筆記，不能寫入或刪除其中內容。  

- **Repo 習慣**  
  - 所有 agent 必須遵守「不直接複製」Issue 原文的原則，僅抽取關鍵概念、決策與未完成事項。  
  - 每日快照 (`daily/*.json`) 只在有可用 Issue 時才產生跨 Issue 主題、決策與開放迴路；若無則保留既有記憶。  
  - 產出之長期記憶 (`MEMORY.md`) 必須是 **精煉、可重用** 的上下文，而非原始日誌或索引頁。  

- **決策與資訊流**  
  - 新的跨 Issue 決策只有在每日快照偵測到「可辨識的跨 Issue 主題」時才會記錄。  
  - 若快照顯示「沒有可用 Issue」，則視為 **資訊暫停**，不新增任何規則或決策。  

## Recent Themes
- **資訊缺乏**：過去七天（2026‑06‑06 至 2026‑06‑12）每日快照皆報告「本次整理視窗沒有可用 issue」，因此目前沒有可辨識的跨 Issue 主題、決策或新興趨勢。  
- **等待更新**：所有快照的「Open Loops」皆指向「等待下一輪 issue 更新後再整理」，顯示目前的工作流正處於待命狀態。  

## Constraints
1. **資料來源限制**  
   - 只能引用 Issue / Comment，不能直接搬錄原始文字。  
2. **手動筆記保護**  
   - `shared/manual.md` 必須保持不被自動流程改寫。  
3. **資訊完整性**  
   - 若資訊不足或相互矛盾，必須在 `MEMORY.md` 中明確標示「不確定」或「缺乏足夠證據」。  
4. **摘要原則**  
   - 只保留長期穩定的規則、近期重複出現的主題、重要限制與未完成事項；避免逐段複製原始日誌。
