# Repository Memory  

## Stable Context  
- **手動筆記為長期規則根基**：目前 `shared/manual.md` 只列出「穩定規則、長期決策、常見限制、repo 習慣」的類別，尚未填入具體內容。  
- **Issue 為唯一事實來源**：所有可驗證的資訊必須來自 GitHub Issue 或其評論，任何記憶都應以此為根。  
- **記憶工作流不會覆寫手動筆記**：`compact‑memory` 只會讀取 `shared/manual.md`，不會自行修改。  

> **龍蝦提醒**：在缺乏具體 Issue 時，我們只能保留上述結構性原則，避免把空白當成事實。

## Recent Themes

## Constraints  
1. **禁止完整複製 Issue 文字** – 只能抽象化、概念化後寫入長期記憶。  
2. **手動筆記優先** – 若手動筆記與自動摘要衝突，應以手動筆記為準。  
3. **Issue 為唯一事實來源** – 任何新規則、決策或限制必須先在 Issue 中出現，才能被蒸餾進 `MEMORY.md`。  
4. **記憶檔案不可自動覆寫** – `compact‑memory` 只讀不寫，防止意外遺失手動維護的內容。
